Data types in JS:

3-Categories:

  • Primitive or primary:
    • Strings:
      • Any grouping of characters on your keyboard (letters, numbers, spaces, symbols, etc.) surrounded by single '' or double "".

    • Number:
      • Any number, including numbers with decimals: 1, -2, 99, 3.14.

    • Boolean:
      • This data type only has two possible values — either true or false.

    • Null:
      • This data type represents the intentional absence of a value, and is represented by the keyword null.
      • Null means there is no value here

    • Undefined:
      • This data type is denoted by the keyword undefined. It also represents the absence of a value though it has a different use than null.
      • Undefined means there should be some values, but it is undefined now

  • Composite or reference:
    • Arrays:
      • Arrays are lists of ordered, stored data. They can hold items that are of any data type.
      • Nested Arrays: Arrays with one or more arrays as elements are referred to as “nested arrays”.

    • Functions:
      • Functions are one of the fundamental building blocks in JavaScript.
      • A function is a reusable set of statements to perform a task or calculate a value.
      • Functions can be passed one or more values and can return a value at the end of their execution. In order to use a function, you must define it somewhere in the scope where you wish to call it.

    • Date:
      • It represents a single moment of a time. When used as a function, Date() returns the current date and time. When used as a constructor, Date() returns a new date object.

    • Objects:
      • Object always declared in 'key-value' pairs.
      • The 'key:value' pairs in JavaScript objects are called properties:
      • Object properties can be accessed using the dot notation: objectName.propertyName
      • Bracket notation must be used if the property name has spaces, special characters, or begins with a number: objectName['property Name']
